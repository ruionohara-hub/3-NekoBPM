# 3-NekoBPM

制作期間 : 約1ヵ月

NekoBPMは、DAWのテンポ(BPM)に完全に同期してキャラクターが踊るプラグインです。音楽制作中の視覚的な楽しさを提供するだけでなく、プロジェクトのテンポ感を直感的に把握することを助けます。

## 技術的な工夫点

本プロジェクトでは、C++のオーディオフレームワーク「JUCE」のポテンシャルを最大限に活かし、高精度なビジュアル同期を実現しました。

* **業界標準フレームワーク「JUCE」の選定とクロスプラットフォーム対応**
    音楽制作環境における高い安定性と、Windows/macOS(VST3/AU)の両対応を考慮し、C++のJUCEフレームワークを採用しました。DAWの再生から正確なテンポ情報（`PositionInfo`）を取得する処理を核としています。
  
* **動的なアニメーション速度制御ロジックの実装**
    単なるループ再生ではなく、DAWのBPMとアニメーションのベースBPM(120BPM)の比率から、フレーム更新間隔をリアルタイムに算出する計算式を実装しました。これにより、テンポを急激に変化させても、キャラクターの動きがズレることなく音楽に追従します。
  
* **BinaryDataによるアセットの一体化と配布効率の向上**
    158枚に及ぶPNGフレームを、JUCEのBinaryDataシステムを用いて実行バイナリ内に埋め込みました。外部の画像パスに依存しないため、プラグインファイルをインストールするだけで確実に動作する、ユーザーに優しい配布形態にしています。

## ユーザー観点での工夫

「楽曲制作をより楽しく、クリエイティブに」という視点から、UI/UXを設計しました。

* **直感的な動作ステータスの可視化**
    DAWが「再生中」か「停止中」かを判定し、キャラクターの動作だけでなくテキスト表示のカラーでも即座に状態が伝わるようにしました。作業中の小さな確認を減らせるようにしました。
  
* **UIの削減とノブの排除**
    複雑な設定ノブを排除し、DAWにロードするだけで機能が完結するように設計しました。画面の端に置いておいても邪魔にならないようにしています。またUIの削減をすることでプラグインを読み込んでもDAWが重くならないようにしています。

## 課題に対するアプローチ

* **大量の画像描画によるCPU負荷の最適化**
    158枚のフレームを毎秒高頻度で切り替える際、描画負荷が制作の妨げにならないよう工夫しました。`juce::Timer`のコールバック内で、必要なタイミングのみ`repaint()`を呼び出し、メモリにロード済みの`std::vector<juce::Image>`から直接描画することで、滑らかなアニメーションと低負荷の両立を実現しました。

## エピソード

* **制作の流れ、きっかけ、得たこと**
    私自身、趣味でアーティストをしており、DAWで楽曲を制作する際にYouTubeでフリーの音源などを探したりするのですが、概要欄にダウンロード先のURLが記載されていない場合があり、そういった時はwebで変換できるサイトなどを使用し、ダウンロードしておりました。その際「webでの広告の煩わしさ」「変換速度が安定していない」などの問題が多く、自身で制作するに至りました。まず意識したのは速度よりも音質です。実際に楽曲制作で音源を使用する際、mp3よりもwavで使用したい方が多いと思ったため、wavに変換するプログラムとしての制作を開始しました。課題が解決する毎に改善点や追加点なども増えていき、ユーザーであり制作者という立場で制作を進められたのは大きかったと思います。得たこととして、長期に及ぶ開発の際に少し期間が開くとファイルを見直したときにソースコードを再度理解しなくてはならない状況が多いことを理解しました。そのため制作の後半では視認性のいいコメントアウトを追加し、現在では長期制作の効率化を行えるようになりました。
